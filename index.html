Hexo 更换电脑后

By 影乐

发表于 2015-10-01

起因：因下载了一个ipv6代理软件，强迫症严重发作。
所以，我必须重装电脑！！！
更换电脑或者重装电脑后，Hexo该这样…

备份

备份hexo目录下的source目录（文章）、_config.yml文件（配置）、themes文件夹（主题）。
备份完成后，就可以重装电脑啦~

安装必备软件

nodejs
git

安装Hexo以及依赖环境

npm install -g hexo-cli -g
cd 到 hexo 目录键入$hexo init
然后npm install安装依赖环境
之后就可以$hexo server看到效果了..

安装插件

npm install hexo-deployer-git --save 部署
npm install hexo-generator-feed --save 订阅
npm install hexo-generator-sitemap --save 网站地图
以上是我所使用的插件，其他插件自行搜索。

还原备份

直接把备份文件替换掉新的就可以了

密钥上传

可以参考最下边的链接生成密钥，我用的是Github Desktop直接生成密钥并上传。
所以我只需更改下Hexo所在的目录就可以了。

上传

然后就可以$hexo d -g部署了
如果无法上传，可以替换成git 或者 https

人生需要不完美

By 影乐

发表于 2015-09-28

本文算是一个最近乱想的一个随笔。

人类的神经有多脆弱

26日放学回家，车站很堵，人山人海。
好不如意做上车，睡了一觉。醒来就听见后边有俩2b在唱歌，整天大客车都能听见。
我醒来的时候已经是傍晚了，天有点暗了，有的阴森森的感觉。
那种感觉也让我有一种即兴发挥，不得不感慨人类的神经真脆弱。
不止神经，生命也一样。珍惜吧，做自己想做的事。

想一个人独自去旅行

下了车，天已经黑了。过马路的时候看到红绿灯，听着这个城市忙碌的声音。
让我愈加想离开山东，到一个陌生的城市独自生活。
可惜自己尚未成年，太多的局限。

万事具有多面性

一直以为男生纹理烫比较帅，其实也只是男生这么认为罢了。
女生可不觉得纹理烫帅，女生喜欢男生的发型简单干净。
既然已经烫了，就先这样吧。全当一次尝试，一次经历。

也谈审美观

好多人说我的审美观有点非主流、杀马特，
“红裤子花衬衫的影乐” “非主流发型”
做事不能平凡，但穿着还是要适当的平凡些。
至于为啥会这样，我也不知道。
反正尝试改变下吧~
已经把博客和QQ头像…做了一些修改，
希望能纠正下我的审美观吧~

人生需要不完美

最近想做的事情太多了，做了之后又想放弃。
想学单片机，又想学Web前端，还有散打，还想玩些网赚…
一时间发现我的爱好好多..却难以坚持。
其实人生只需专注一项，认真的做下去就好了。
其他爱好发展发展，当作业余技能，这样挺好。

成长和初恋以及未来

By 影乐

发表于 2015-09-12

文章最下方音乐会自动播放，半夜看文章的注意啦~~

开学之后

新学期已经过了一周了，从满怀期待到释然经历了太多。
开学的那天是周日，我和同学早晨从家到车站，找了个川菜馆子吃了饭就去学校了。
一路上很开心，毕竟在家挺无聊的，终于能得到释放了。
新学期，我们和另一个专业的班级合班了，据说这是每年都会发生的事。
合班之前，我们以为会很难相处，但事实并非如此。
一切还是和往常一样，该吃吃该喝喝。
合班之后，自然要调位，要让大家没有班级的间隔，融为一体。
可她，因为不想换同桌而哭了…
虽然我们早已分手，但还是心里觉得有点难受。
甚至想骂她！可我算个“What”？
当天晚上我就问了班主任，可不可以挂学籍。
班主任可能很忙，就跟我说“不可以，这是没有的事”，我也就没再继续说什么。
大概第二天晚自习，班主任找我到她的新办公室，跟我谈了很多。
我也跟她说了很多退学的人的故事，但她告诉我，
可那只是个别例子，你退学后干什么？
一年都混过来了，再混一年拿个毕业证多好啊，以后还可以去当兵。
道理都懂，我也没再继续说什么。不退学算是答应父母的要求，
而且我退学后，也确实没有能力养活自己。

不忘初心 做你所想

By 影乐

发表于 2015-09-04

小时候，我们都会怀有一个梦想。随着年龄的增长，我们的梦想越来越小，或者说是更现实了。甚至丢了梦想，只想平平淡淡地生活。每个人都有自己的追求，平平淡淡地生活也挺好。可我不想，我是一个不甘愿平凡的少年。但不甘愿平凡又指什么呢？百科给我们的解释是指平常；普通，平庸是它的近义词，而出众是它的反义词。我想不甘愿平凡大概是指不想平庸，想出众。就是鹤立鸡群，万众属目。或者说是“另类”。

但我还是做着寻常的事。我觉得没有必要刻意的去做不寻常的事，因为没有必要且不值得那样做。我只想心怀梦想努力向上就行了，不要和他人攀比。他人不是你，你是独一无二的。超越自己才是进步，就如刚出生的你，到现在已经学会了很多技能。你已经超越了那个自己，可你要拿别人当作目标，只会让你失去“独一无二”。

我们在追寻梦想的道路上，总会跑偏。就如童年的我们，游戏玩不好，就想着找外挂。最后把精力都投入到找外挂上面了，到头来二场空。也会在追寻梦想的道路上，积压太多包袱。拿得起，放不下。太在意这些虚拟的东西，道理都懂，可终究还是放不下…

试着一点一滴地放下包袱吧！去寻找你真正的梦想，把它划分成很多小目标。去实现，去追求。永远要记住，今天的你必须超越昨天的自己！

VPS如何搭建WordPress

By 影乐

发表于 2015-09-03

闲着没事想翻墙，就买了一个搬瓦工，$6.99/月。ss的速度在我这里，白天能达到1M+/s 1080P无压力。
晚高峰有点慢，但浏览网页还是无压力的。

登录到vps

使用Xshell等工具登录你的vps

更新

#apt-get update

安装所需环境Apache2 mysql php5以及扩展

安装Apeche2#apt-get install apache2
安装MySQL#apt-get install mysql-server密码需要记住！
安装PHP5#apt-get install php5

继续安装扩展#apt-get install php5-mysql 否则会出现（您的 PHP 似乎没有安装运行 WordPress 所必需的 MySQL 扩展错误）

下载WordPress,并解压

#cd /var/www/html
#wagt https://cn.wordpress.org/wordpress-4.3-zh_CN.tar.gz

解压tar.gz #tar xvfz wordpress-4.3-zh_CN.tar.gz
如果你下载的是zip,那就使用#unzip wordpress-4.3-zh_CN.tar.gz

进入WordPress编辑配置

#cd /var/www/html/wordpress
编辑配置#vi wp-config-sample.php

/ MySQL数据库名 */
define(‘DB_NAME’, ‘wordpress’);
/ MySQL数据库用户名 /
define(‘DB_USER’, ‘root’);
/** MySQL数据库密码 /
define(‘DB_PASSWORD’, 密码’)
/* MySQL主机（保持原样就行） /
define(‘DB_HOST’, ‘localhost’);

修改完后重名为wp-config
#mv wp-config-sample.php wp-config.php

重启Apeche和MySQL

重启Apeche2#/etc/init.d/apache2 restart
重启MySQL#/etc/init.d/mysqld restart

新建数据库并修改mysql权限

修改文件路径权限#chomd 777 /var/lib/mysql
#mysql -u root -p输入密码
新建数据库mysql>create database wordpress;

新增WWW的数据写入权限

#chown -R www-data:www-data /var/www/html/wordpress

解析域名

现在Apeche2不依靠httpd.conf来配置。
Ubuntu下默认的配置文件路径在/etc/apache2/sites-available/default
进入配置路径#cd /etc/apeche2/sites-available
配置#vi 000-default.conf
修改文件路径为/var/www/html/wordpress/
然后把域名解析到你的vps提供商提供给你的ip

安装WordPress

访问你的域名，进行5分钟安装WordPress

从Wordpress到Hexo

By 影乐

发表于 2015-08-24

容我废话几句。

2015/8/24 因技术不到位，友联迁移到内页，希望友情不会因此结束
2015/8/26 更换主题，友联重回首页

目前还处于折腾期，sitemap和xml我弄不出来。。。插件安装好了，也设置好了，就是不出来！！

折腾了二天，终于从Wordpress迁移到了Hexo..
不过我那888条评论弄不回来了，因为我不会！！呜呜呜 ~~(>_<)~~
如果不是因为Wordpress太慢了我也不想换啊::>_<::

另外更换404页面为公益404，点这里一起寻找丢失的孩子吧~

记录一下折腾的笔记吧~

1.安装环境

Git客户端
Node.js

2.ssh密钥

$cd ~/. ssh检查本机的ssh密钥
第一次使用会提示No such file or directory
接着在Git客户端中键入 $ssh keygen -t rsa -C "你的邮箱"生成ssh密钥

3.上传密钥

使用Notepad++等编辑器复制ssh.pub里的内容，粘贴到github的ssh密钥管理中。
ssh.pub一般在C:\用户\admin.ssh

4.测试设置是否成功

$ssh -T git@github.com输入yes之后接着输入密钥的密码
之后就会看到 Hi 你的用户名! You’ve successfully authenticated,
but GitHub does not provide shell access.
说明设置成功了

5.设置用户信息

$git config --global user.name "用户名"
$git config --global user.email "你的邮箱"

6.在Github中创建一个新的存储库

名字需要注意，要省事的话直接名为你的Github用户名.github.io

7.安装Hexo

$npm install -g hexo-cli
然后新建一个文件夹，在文件夹内右键打开Git Bash
键入$hexo init 初始化程序
然后继续键入$npm install安装依赖环境
之后键入$hexo sever打开localhost:4000就可以看到Hexo了!

8.部署

部署之前先配置_config.yml
在里边添加

deploy:
  type: git
  repo: <你的仓库地址,可以是https 也可以是git>
  branch: master

保存之后，键入$hexo d -g
就可以完成部署了
每次部署前最好$hexo clean清理下缓存

0.部分说明

hexo new [layout] 文章标题 #写文章，可简写为hexo n
hexo generate #生成 ， 简写为hexo g
hexo deploy #部署 ， 简写为hexo d

Chrome 报错 ERR_CONNECTION_ABORTED

By 影乐

发表于 2015-08-16

Firefox更新后，Flash老是崩。就转战Chrome了，不是别的浏览器不够好。是因为我有选择强迫症，在我眼里Windows下的浏览器只有火狐和扣肉。
可最近不知道怎么回事，Chrome下不能访问https://www.baidu.com 等https链接
其他浏览器正常。在贴吧也看到一个人发帖也是出现这种情况。
下面是解决方法：

更换DNS

更换了DNS就ok了…也不知道为啥，为啥只有Chrome受影响，其他浏览器没事。

清理DNS缓存

2015.8.24 update…
无意中发现Chrome有个内置管理DNS的功能
地址在chrome://net-internals/#dns
麻麻再也不用担心我用完代理之后打不开百度啦~

允许谷歌更新程序自动启动

2015.8.28 update…
还是继续出现访问异常的情况，然后把谷歌更新程序、谷歌自动更新任务从加速任务中忽略。
第二天访问一切正常了。

从zhidao.baidu.com到达百度

百度首页无法访问，可以先进入zhidao.baidu.com，然后点击网页，就能打开百度了。

和插件有冲突

尝试隐身模式下访问百度，然后排查是那个插件出现问题。

其他解决办法

至此，这个错误还没有得到解决。多数出现在使用代理后。
Chrome 错误 103 是一个未知错误，所以解决方法并不适应所有用户。
期待留下你的解决方法，后期我会补在这篇文章里。
